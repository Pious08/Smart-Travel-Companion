# Smart Travel Companion User Stories

**1. Login & Registration Screen**

  **US1 (Registration):** As a new traveler, I want to register by entering my username, email, and password and clicking Sign Up so that I can create a personal travel profile.
    
  **US2 (Login):** As a returning traveler, I want to input my email and password and click Login so that I can access my saved destinations.
    
  **US3 (Feedback):** As a user, I want to see an error message if I leave fields blank or enter the wrong password so that I know how to fix my input.
    
  **US4 (Persistence):** As a user, I want my credentials stored in local storage (Shared Preferences) so I stay logged in even if I close the app.

**2. Home Screen**

  **US1 (Overview):** As a user, I want to see a list of my "Pinned Destinations" on the home screen to monitor my upcoming trips at a glance.
  
  **US2 (Intro Guide):** As a new traveler, I want to see a "Quick Start" carousel explaining how to add my first destination.
  
  **US3 (Quick Actions):** As a user, I want shortcuts to "Add New Trip" and "Open Maps" directly from the home screen for efficient navigation.

**3. Detail Screen**

  **US1 (Details):** As a traveler, I want to see images, descriptions, and safety ratings for a selected city to make informed travel plans.
  
  **US2 (Share/Save):** As a traveler, I want to share a destination's details with my friends via social media directly from the app.
  
  **US3 (Related):** As a traveler, I want to see "Nearby Attractions" listed at the bottom of a destination page to explore more options.

**4. Data Persistence**

  **US1 (State):** As a user, I want my login state to persist across sessions so I don't have to re-login every time I open the app.
  
  **US2 (Preferences):** As a traveler, I want the app to remember my choice of Dark Mode or Metric units ($Celsius$ vs $Fahrenheit$).
  
  **US3 (Logs):** As an admin, I want to track which destinations are added most frequently to analyze travel trends.
  
**5. External API Integration**

  **US1 (Weather API):** As a user, I want to view real-time weather updates for my saved cities so I can pack appropriate clothing.
  
  **US2 (Currency API):** As a traveler, I want to see live exchange rates for the local currency of my destination.
  
  **US3 (Maps API):** As a traveler, I want to see my saved hotels on a map to choose the most convenient dining locations nearby.

**6. Settings Menu**

  **US1 (Access):** As a user, I want to access the Settings via a gear icon in the app bar from any screen.
  
  **US2 (Categorization):** As a user, I want to see settings grouped into Profile, Appearance, and Privacy for quick discovery.
  
  **US3 (Admin):** As an admin, I want to toggle the "Beta Features" visibility for specific users to test new updates.

**7. Settings Screen**

  **US1 (Dark Mode):** As a user, I want to enable Dark Mode to save battery and reduce eye strain at night.
  
  **US2 (Notifications):** As a user, I want to toggle travel alerts on or off to avoid unnecessary interruptions.
  
  **US3 (Account Update):** As a user, I want to update my email or password in the settings to keep my account secure.

**8. Notifications**

  **US1 (Reminders):** As a user, I want a daily notification reminding me to check my flight status or itinerary.
  
  **US2 (Feature Updates):** As a user, I want to be notified when a new city guide is added to the app.
  
  **US3 (Control):** As a user, I want to disable promotional notifications while keeping travel alerts active.
  
  **US4 (Targeting):** As an admin, I want to send a "Special Discount" notification only to users who have "Japan" in their bucket list.
